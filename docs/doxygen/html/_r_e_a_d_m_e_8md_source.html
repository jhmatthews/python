1 # Python
2 
3 Python is a (confusingly named) Monte Carlo radiative transfer code which uses the Sobolev approximation.
4 It has been developed by Knox Long, Christian Knigge, Stuart Sim, Nick Higginbottom, James Matthews and Sam Mangham.
5 The code has been used for a variety of research projects invovling the winds of cataclysmic variables, of young stellar
6 objects, and of AGN.
7 
8 The code is under active development, but we are looking for beta users to test the code, and potentially use it
9 for their own research. If you are interested in using Python please email the collaboration at agn-winds[at]googlegroups[dot]com or contact Knox Long via long[at]stsci[dot]edu.
10 
11 Documentation of input parameters is hosted on [ReadTheDocs](http://agnwinds.readthedocs.io/en/dev/).

17 ## Installation
18 
19 Python and the various routines associated are set up in a self-contained directory structure. The basic directory structure and the data files that one needs to run Python need to be retrieved and compiled.
20 
21 If you want to obtain a stable (!) release, go to the [Releases](https://github.com/agnwinds/python/releases) page.
22 
23 If you want to download the latest dev version, you can zip up the git repository by clicking on the zip icon to the right of the GitHub page. Aternatively, you can clone the repository using
24 
25  $ git clone https://github.com/agnwinds/python.git
26 
27 If you anticipate contributing to development we suggest Forking the repository and submitting pull requests with any proposed changes.
28 
29 Once you have the files, you need to cd to the new directory and set your environment variables
30 
31  $ export PYTHON = /path/to/python/
32  $ cd $PYTHON
33  $ ./configure
34  $ make install
35  $ make clean
36 
37 note that export syntax is for bash- for csh use
38 
39  $ setenv PYTHON /path/to/python/
40 
41 Atomic data is stored in our [data repository](https://github.com/agnwinds/data) with it's own releases page. one should unzip these files and place them in a $PYTHON/data folder.
42 
43 A development user may want to work on atomic data as part of their work, and pull in changes as they are made, in which case we recommend cloning the data repository:
44 
45  $ cd $PYTHON; git clone https://github.com/agnwinds/data data
46 
47 ## Running python
48 
49 To run python you need to add the following to your $PATH variable:
50 
51  $PYTHON/bin
52 
53 You can then setup your symbolic links by running
54 
55  $ Setup_Py_Dir
56 
57 and run the code by typing, e.g.
58 
59  $ py root.pf
60 
61 
62 Please see the [wiki](https://github.com/agnwinds/python/wiki/Installing-and-Running-Python) and docs folder for how to use the code.
