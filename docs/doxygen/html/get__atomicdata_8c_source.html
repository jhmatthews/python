26 //OLD get_atomic_data(masterfile) is a generalized subroutine for reading atomic data

34 //OLD as atomic data. (This allows one to group the atomic data into logical units.)
35 //OLD
36 //OLD Returns:
37 //OLD
38 //OLD
39 //OLD Description:
40 //OLD get_atomic_data reads in all the atomic data. It also converts the data to cgs units unless
41 //OLD otherwise noted, e.g ionization potentials are converted to ergs.
42 //OLD
43 //OLD The order of the data is important. Elements should be defined before ions; ions
44 //OLD before levels, levels before lines etc. In most if not all cases, one can either define all
45 //OLD of the elements first, all of the ions, etc ... or the first element and its ions, the
46 //OLD second element and its ions etc. Commenting out an element in the datafile has the effect of
47 //OLD eliminating all of the data associated with that element, e.g ions, lines, etc.
48 //OLD
49 //OLD The program assumes that the ions for a given element are grouped together,
50 //OLD that the levels for a given element are grouped together, etc.
51 //OLD
52 //OLD If one wants to read both Topbase and VFKY photoionization x-sections, then the topbase
53 //OLD x-sections should be read first. The routine will ignore data in the VFKY list if there
54 //OLD is a pre-existing Topbase data. The routine will stop if you but a VFKY x-section first
55 //OLD on the assumption that Topbase data should trump VFKY data. (Making the program more flexible
56 //OLD would be difficult because you would have to drop various bits of Topbase data out of the
57 //OLD middle of the structure or mark it NG or something.)
58 //OLD
59 //OLD Similarly, as a general rule the macro information should be provided before the simple
60 //OLD ions are described, and the macro lines should be presented before simple lines.
61 //OLD
62 //OLD The only data which can be mixed up completely is the line array.
63 //OLD
64 //OLD Finally, get_atomic_data creates a pointer array to lines, which has the lines in
65 //OLD frequency ascending order. This is actually done by a small subroutine index_lines
66 //OLD which in turn calls a Numerical Recipes routine.
67 //OLD
68 //OLD Notes:
69 //OLD NOTHING IN THESE ROUTINES SHOULD SPECIFIC TO THE WAY IN WHICH THE ATOMIC DATA
70 //OLD IS TO BE USED. Specifically nothing here should be specific to the Monte Carlo calculations
71 //OLD for which the routines were written. These routines are just ways to read in the atomic data
72 //OLD
