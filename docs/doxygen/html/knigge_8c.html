Detailed Description

Routines needed to define a KWD wind.

Author
ksl
Date
January, 2018

The routines here are all that are needed to define a KWD-type wind within Python. Inputs are gathered, and the any values such as the nomalization needed to convert a global to a local mass loss rate. Functions are provided to calculated the density and velocity of the wind at any point.

As written, the routines assume a standard Shakura-Sunyaeve disk temperature distribution.

Although a KWD wind is only really defined within wind cones, the routines will return a non-zero velocity and density outside these regions to allow for interpolating at the edges of the wind cones.

double kn_wind_mdot_integral ( double  r)

the integrand of KWD model for mdot as a function of radius

Parameters
[in]doubler A position (radius) in the disk
Returns
The value of the integrand

The mass loss rate is proportional to T(r)**(4*kn_lambda) as can be seen from KWD95 equation 10. This functikon has to be integrated over radius to normalize the local mdot to the integrated value.

Notes

The integration is carrid out in get_knigge_wind_params

There is an extra factor of 2 because the wind emerges from both sides of the disk
