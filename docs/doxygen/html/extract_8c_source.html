24 //OLD int extract(w,p,itype) is the supervisory routine which helps normally
25 //OLD builds detailed spectra as the photons are transit the wind.
26 //OLD
27 //OLD Arguments:
28 //OLD PhotPtr p; The initial photon
29 //OLD WindPtr w;
30 //OLD int itype
31 //PTYPE_STAR->the photon came for the star
32 //OLD PTYPE_BL->the photon came from the boundary layer
33 //OLD PTYPE_DISK->the photon being redirected arose in the disk,
34 //OLD PTYPE_WIND->the photon being redirected arose in the wind,
35 //OLD Returns:
36 //OLD
37 //OLD
38 //OLD Description:
39 //OLD
40 //OLD extract is called when a photon begins it's flight and every time that photon
41 //OLD scatters, unless the user has exercised the "live or die" option, in
42 //OLD which case it is not called. extract calls extract_one for each spectrum
43 //OLD it wants to build, where the actual incrementing of the spectrum is done.
44 //OLD
45 //OLD An option allows one to construct spectra which have undergone a certain
46 //OLD number of scatters. The parameters for this option all come in through
47 //OLD python.h.
48 //OLD If s[n].nscat>999; then one obtains the entire spectrum
49 //OLD if s[n].nscat is a positive number, then the spectrum is composed just
50 //OLD of photons with that number of scatters
51 //OLD if s[n].nscat is a negative number, then the spectrum contains all photons
52 //OLD with >= the absolute value of s[n].nscat
53 //OLD Another option allows one to construct spectra from photons which have
54 //OLD scattered above or below the plane of the disk. As for the number of
55 //OLD scatters this option comes through python.h, in this case from s[n].top_bot
56 //OLD s[n].top_bot=0 -> accept all photons
57 //OLD >0 -> accept photons last above the disk
58 //OLD <0 -> accept photons below the disk
59 //OLD Notes:
60 //OLD
61 //OLD History:
62 //OLD 97march ksl Coded and debugged as part of Python effort.
63 //OLD 97aug28 ksl Added the option which allows one to construct spectra with a
64 //OLD specific number of scatters.
65 //OLD 97sep1 ksl Added the option which allows one to construct spectra which
66 //OLD originate from a specific position above or below the disk
67 //OLD 02jan ksl Fixed problems with itypes. (Photons which were wind photons
68 //OLD for the purpose of extract were not being properly labelled
69 //OLD as such, and this meant they were not doppler shifted properly.
70 //OLD This produced photons that were further from resonance in the
71 //OLD local frame and made the wind more transmissive, especially
72 //OLD in the blue wing of the line. It suggests
73 //OLD that a lot of the absorption in extract is fairly local and
74 //OLD that the blue wing structure could be fairly sensitive to how
75 //OLD the doppler shift is carried out.)
76 //OLD 09feb ksl 68b - Added hooks to track energy deposition of extracted photons
77 //OLD in the wind
78 //OLD 15aug ksl Modifications to allow multiple domains.
