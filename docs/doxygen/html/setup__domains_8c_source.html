200  * we wish to create, say a spherical outflow with two domains one going from
201  * r1 to r2 and the other going from r2 to r3. Then we want to keep geo.rmax which is
202  * intended to be the distance beyond which photons are moving through free space separate
203  * from the values in the wind zones. Right now we are setting the outer limit of each
204  * wind to be geo.rmax regardless, in routines like get_stellar_wind_params and get_sv_wind

206  * relevant we should ask for the outer edge of the domain and then at the end we should determine
